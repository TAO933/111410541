

## 使用Google Antigravity

**本作業的程式碼與結構是透過 Google Antigravity生成的。**



## 功能重點整理
- **基礎與進階要求**
  - **單一輸入區**：將兩個輸入框改為仿照手機/電腦小算盤的單一 TextBox (本專案使用 TextBlock 達更佳排版效果)。
  - **直覺介面**：提供完整的 0-9 數字鍵與四則運算符號。
  - **回刪功能 (Backspace)**：允許刪除最後輸入的單一字元，避免填錯需要全部重來(`⌫` 鍵)。
  - **清除功能 (C/CE)**：提供 C (清除所有狀態) 與 CE (僅清除當前輸入值)。
- **加分項目 (Bonus)**
  - **介面美化**：整體採用 Modern UI 深色主題，按鈕具備隨滑鼠互動的變色動畫 (Hover & Pressed 觸發器)。
  - **鍵盤支援**：除了滑鼠點擊外，更支援直接使用實體鍵盤的數字區 (NumPad)、四則運算符號、Enter (等號)、Backspace、Escape (清除) 進行純鍵盤操作。
  - **進階運算**：結合常用的計算機按鈕，支援了倒數 (1/x)、平方 (x²)、開根號 (√x) 與百分比 (%) 的快捷轉換。
- **例外與防呆處理 (防 Bug)**
  - 當使用者觸發「除以零」或「對負數開根號」時，系統不會崩潰跳錯，而是攔截 `DivideByZeroException` 等錯誤，在畫面上友善提示「無法除以零」。
  - 防呆機制包含：避免輸入超過兩個小數點、限制輸入長度。

## 程式碼架構解析

 <!-- 基本按鈕樣式：應用於數字鍵 -->
**狀態變數定義**

 ```xaml.cs            
// previousValue: 用來儲存運算符號左邊的數值 (例如 A + B 中的 A)
private double previousValue = 0;

// currentOperator: 用來儲存當前點選的運算符號 (例如 +, -, *, /)
private string currentOperator = "";

// isNewInput: 狀態旗標。判斷下次點擊數字時，是否應該重新開始輸入新數字 (取代目前畫面的數字)
private bool isNewInput = true;

// isErrorState: 錯誤旗標。用來處理像是「除以零」的例外狀況，避免程式崩潰
private bool isErrorState = false;

```
- `MainWindow.xaml`: 負責使用者介面。
  - `<Window.Resources>`：定義所有的按鈕 `Style`，包含預設顏色、圓角 `CornerRadius` 與 `Trigger` (處理滑鼠互動)。
  - `<Grid>`：將視窗切分為 3 列 (Row)：歷史紀錄區、主數字區、以及用 6x4 網格排列的按鈕區。
- `MainWindow.xaml.cs`: 負責後端運算邏輯。
  - **狀態機**：使用 `previousValue` (上一個運算元)、`currentOperator` (準備執行的符號)、`isNewInput` (是否該開啟新數字輸入段落)、`isErrorState` (是否剛發生例外錯誤) 來管理計算機的狀態。
  - **事件處理**：利用 `Number_Click` 等方法來對應 Button Click 事件，並包含核心的 `CalculateResult` 方法來進行加減乘除。
  - **鍵盤事件**：透過綁定 `Window_KeyDown` 來攔截並模擬滑鼠點擊。
