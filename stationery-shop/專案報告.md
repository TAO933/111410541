# 一頁式文具店 (Stationery Shop) - 專案報告

## 1. 專案簡介
本專案為一個「高質感、一頁式」的線上文具店。採用現代化的前端技術與後端架構，達成從商品檢視、訂單成立、到商品評論的一條龍完整流程，並且所有操作皆不須切換頁面（SPA 模式），以提供最佳且最流暢的使用者體驗。

## 2. 系統架構與技術選型
*   **前端 / 後端框架：** Next.js (使用 App Router 模式)
    *   **選擇原因：** 能夠透過 API Routes 將前端（介面）和後端（伺服器）完美整合於一個專案內，且 Server Actions 或原生的 API 皆能夠大幅減輕處理非同步請求的難度。
*   **資料庫：** SQLite (better-sqlite3)
    *   **選擇原因：** 適合作為輕量級的後端資料庫，無需額外啟動獨立伺服器與設定密碼連線，極大降低了環境部署的門檻。
*   **介面樣式：** Vanilla CSS / Google Fonts
    *   **設計風格：** 捨棄預設生硬的版面配置，全盤導入高端的 **Glassmorphism (玻璃擬物化)**，輔以流暢的過場漸變（Gradients）、Hover 微動畫，打造符合精緻文具風格的高端介面。使用 Inter 與 Noto Sans TC 確保中英文排版一致性。

## 3. 功能模組說明

### (1) 商品展示 (Product Grid)
*   首頁具備吸睛的 Hero Section 視覺。
*   下方列出文具產品資料集，共計 **30 項真實來自九乘九購物網的精選商品**。
*   商品依據種類被分為三大系列展示：**書寫與修正系列**、**收納與整理系列**、**膠貼與辦公系列**，讓顧客瀏覽體驗更有層次。
*   資料載入具備脈衝 (Pulse) 動畫提示載入狀態。

### (2) 購物車預覽與結帳 (Order Modal)
*   點擊「查看 / 購買」後，會以不換頁方式開啟具有模糊背景 (Backdrop blur) 的懸浮視窗。
*   使用者可直接在畫面中填寫姓名與數量。
*   點選「結帳」後，表單將以 POST 方法傳送至 `/api/orders`，由後端 Server 將訂單金額與數量一併寫入至 SQLite 內的 `orders` 資料表保存。
*   附帶完成狀態的高質感成功提示。

### (3) 評論與回覆留言板 (Comment Section)
*   **讀取與寫入**：與結帳視窗整合，允許使用者針對特定商品留下意見。
*   **巢狀結構**：支援回覆功能，當留言附帶 `parent_id` 時會被視為第二層的回覆，這充分展示了關聯式資料庫的外鍵特性應用。

## 4. 資料庫綱要 (Schema)
整個系統包含三個核心資料表：
1.  **products**: 紀錄商品名稱 (`name`)、價格 (`price`)、敘述 (`description`) 與展示圖片 (`image`)。
2.  **orders**: 紀錄下單的商品 ID (`product_id`)、購買數量 (`quantity`)、總價 (`total_price`)、買家名稱 (`customer_name`) 與日期。
3.  **comments**: 紀錄商品 ID (`product_id`)、留言者 (`author_name`)、內容 (`content`)、回覆關聯 (`parent_id`) 與日期。
