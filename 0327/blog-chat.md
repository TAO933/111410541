# Blog 系統開發 - AI 對話紀錄

## 專案概述

本專案使用 Node.js + SQLite 開發一個簡易網誌系統，包含多個版本的主題設計。

---

## 開發歷程

### 1. 初始版本 - 基礎網誌系統

**建立時間：** 2026年3月27日

**功能：**
- 查看所有文章
- 發布新文章（標題 + 內容）
- 刪除文章
- SQLite 資料庫儲存

**技術架構：**
- Node.js + Express
- sql.js（純 JavaScript 的 SQLite）
- 單頁 HTML 介面

**資料庫結構：**
```sql

  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP

```

## 技術細節

### 資料庫結構（最終版）

```sql
-- 貼文資料表
CREATE TABLE posts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT,
  content TEXT NOT NULL,
  author TEXT DEFAULT '使用者',
  is_public INTEGER DEFAULT 0,  -- 0: 個人, 1: 公開
  likes INTEGER DEFAULT 0,
  comments TEXT DEFAULT '[]',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 個人資料表
CREATE TABLE profile (
  id INTEGER PRIMARY KEY,
  username TEXT DEFAULT '使用者',
  bio TEXT DEFAULT '這是我的個人簡介',
  avatar_color TEXT DEFAULT '#1877f2'
);
```

### API 端點

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/` | 首頁（HTML） |
| POST | `/api/posts` | 發布新文章 |
| DELETE | `/api/posts/:id` | 刪除文章 |

### 啟動方式

```bash
# Threads 版本
cd blog
npm start
# 開啟 http://localhost:3000

# Facebook 版本
cd "blog 1"
npm start
# 開啟 http://localhost:3001
```

## 開發筆記

### 遇到的問題與解決

1. **better-sqlite3 編譯失敗**
   - 原因：環境缺少 Python
   - 解決：改用 `sql.js`（純 JavaScript 實現）

2. **Port 3000 被佔用**
   - 解決：使用 PowerShell 找到 PID 並終止程序

3. **HTML 字串模板複雜**
   - 解決：將 HTML 片段提取為獨立函數，減少模板嵌套
